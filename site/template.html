The agent

How this was researched

A four-stage pipeline (TypeScript, ~600 lines). Every stage checkpoints per app and is re-runnable in isolation — the repo README reproduces this page end to end.

Pass 1
Research agent

One Exa /answer call per app — web search + synthesis in a single request, locked to a strict JSON schema: auth, access, API surface, MCP, verdict, evidence URLs. Closed enums keep the results clusterable.

Cross-check
Composio registry

The Composio SDK + v3 API pulls the full toolkit catalog; each app is matched to see if a toolkit already exists and what auth Composio itself records for it.

Pass 2
Evidence verifier

Layer 1 re-fetches every cited URL — dead citations surface here — and keyword-corroborates each claim against the page text. Layer 2 re-asks every unsettled field as a narrow, schema-locked question; a disagreement overturns the field and is logged.

Pass 3
Human review

A random sample hand-checked against live docs, plus every app the verifier flagged. Human corrections outrank the machine ones.

Output
This page

Merge applies corrections (human > verifier > agent), tags blockers, computes the accuracy movement, and injects everything into one self-contained HTML file.

Where a human was needed

  • Engine pivot. Pass 1 started on Gemini + Google Search grounding; its free tier capped at ~20 requests/day after 16 apps. A human decided to pivot to Exa /answer — and kept the 16 Gemini records as an independent cross-check instead of discarding them.
  • Prompt contamination. The first Exa query said "wrapping it as a tool for AI agents", which steered search toward vendors' AI-marketing pages — Salesforce came back judged on Agentforce docs (one citation was Quizlet flashcards). Caught by eyeballing record #1; fixed by rewording the query around the core developer API.
  • Identity disambiguation. Small or ambiguous names defeated the agent quietly: "Paygent Connect" was assembled from other vendors' payment docs, Waterfall.io's evidence cited an Apollo.io feature, Consensus's cited goconsensus.com (a different product). Only a human noticed the names didn't match.
  • Registry matching. Composio's list API silently caps at 1000 of 1,047 toolkits — a human added cursor pagination, slug probing, and the judgment call that Composio's unified zoho suite toolkit counts as Zoho CRM coverage.
  • Taxonomy judgment calls. Local CLI tools (Mermaid CLI, Sherlock) got "buildable: yes" from the agent — true only as local skills, not hosted API toolkits. Sales-led "usage-based" platforms (Waterfall, PitchBook) needed a human to call them partner-gated.
  • Confidence was noise. The agent rated its own confidence "high" on 100 of 100 apps. A human replaced self-reported confidence with verification-derived per-field checks.

Run facts

  • ~600 lines of TypeScript, two runtime dependencies (the Composio SDK and tsx). Exa and Gemini are called with plain fetch — every request is inspectable.
  • Cost of the whole study: ≈ $2 in Exa credits (pass 1: $0.49 for 100 apps; pass 2 re-asks: ~$1.30). Zero paid app accounts were needed — gated-with-evidence is a valid finding, not a failure.
  • Pass 1: ~13 minutes for 100 apps; pass 2: ~35 minutes (it politely re-fetches every cited page). Everything checkpoints per app and resumes.
  • 16 apps researched twice — once by Gemini+Google, once by Exa — giving a measurable two-engine agreement rate.
  • Reproducible: npm run research && npm run composio && npm run verify && npm run build-data && npm run site rebuilds this exact page from scratch.
